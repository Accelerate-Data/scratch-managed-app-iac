# Release Notes — v0.5.0 (2026-01-05)

## RFC-71 Deterministic Naming Implementation

Implements RFC-71 deterministic naming conventions across all PRD-30 resources using the centralized `iac/lib/naming.bicep` helper.

### Changes

#### Naming Helper Extension
- Extended `iac/lib/naming.bicep` with 34 new name generators:
  - **11 private endpoint names**: `peKv`, `peStBlob`, `peStQueue`, `peStTable`, `peAcr`, `peAppApi`, `peAppUi`, `peFunc`, `peSearch`, `peAi`, `peAutomation`
  - **11 private DNS zone group names**: `peKvDns`, `peStBlobDns`, `peStQueueDns`, `peStTableDns`, `peAcrDns`, `peAppApiDns`, `peAppUiDns`, `peFuncDns`, `peSearchDns`, `peAiDns`, `peAutomationDns`
  - **12 diagnostic setting names**: `diagKv`, `diagSt`, `diagAcr`, `diagAppApi`, `diagAppUi`, `diagFunc`, `diagAgw`, `diagPsql`, `diagSearch`, `diagAi`, `diagAutomation`, `diagLogic`
- All names follow RFC-71 pattern: `vd-{rtype}-{purpose}-{nanoid}` (or `vd-diag-{rtype}-{nanoid}` for diagnostics)
- Nanoid lengths: 16 chars for resources, 8 chars for storage accounts (per RFC-71)

#### Module Updates
All modules updated to use naming parameters instead of hardcoded names:
- `security.bicep`: Private endpoints, DNS zone groups, diagnostic settings
- `compute.bicep`: Private endpoints, DNS zone groups, diagnostic settings
- `ai.bicep`: Private endpoints, DNS zone groups, diagnostic settings
- `automation.bicep`: Private endpoint, DNS zone group, diagnostic setting
- `gateway.bicep`: Diagnostic setting
- `data.bicep`: Diagnostic setting
- `logic.bicep`: Diagnostic setting

#### Entrypoint Fixes
- `main.bicep`: Updated to pass naming outputs to all modules
- `main.rg.bicep`: Fixed to pass naming outputs to all modules (was missing in initial implementation)

#### Repository Cleanup
- Updated `.gitignore` to exclude Bicep build artifacts (`*.json` files generated by `az bicep build`)
- Removed previously tracked JSON build artifacts from git

### Verification

- ✅ Bicep build: All modules build successfully (`main.bicep` and `main.rg.bicep`)
- ✅ Bicep lint: Only expected `nodeSize` unused warning (documented in AD-58)
- ✅ Parameter validation: Tests passing
- ✅ All acceptance criteria verified:
  - All PRD-30 resources use `iac/lib/naming.bicep` helpers
  - Nanoid lengths match RFC-71 (RG: 8, resources: 16, storage: 8)
  - rtype prefixes match RFC-71 table

### Impact

- **Deterministic naming**: All resources now use consistent, deterministic naming patterns per RFC-71
- **Idempotency**: Naming remains deterministic across deployment runs
- **Maintainability**: Centralized naming logic in `iac/lib/naming.bicep` simplifies future updates
- **Compliance**: Full RFC-71 naming standard compliance across all PRD-30 resources

### Migration Notes

No migration required. This change is backward compatible and maintains idempotency. Existing deployments will continue to work, and new deployments will use the updated naming conventions.
